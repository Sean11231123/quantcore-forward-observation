Issue Fixed

The V12 Forward Observation workflow commit-back step could emit/fail around:

fatal: pathspec 'logs/runtime_errors.log' did not match any files

The same failed workflow also saw a GitHub push rejection with Internal Server Error.

logs/runtime_errors.log is optional. Valid no-signal V12 forward observation runs commonly do not create a runtime error log.

Files Changed

- .github/workflows/v12-forward-observation.yml

Exact Fix

The commit-back step now:

1. Always attempts to stage:
   - logs/v12_signals.csv
   - logs/v12_observation_heartbeat.csv

2. Adds logs/runtime_errors.log only if it exists:
   if [ -f logs/runtime_errors.log ]; then
     git add logs/runtime_errors.log
   else
     echo "logs/runtime_errors.log not present; skipping optional runtime error log"
   fi

3. Exits cleanly if there are no staged changes:
   git diff --cached --quiet

4. Creates at most one commit per workflow run.

5. Retries git push up to 3 times with 10 seconds between attempts.

Why Push Retry Does Not Create Duplicate Commits

The workflow creates the commit once before the retry loop. The retry loop only retries pushing the same commit. It does not run git commit again inside the loop.
