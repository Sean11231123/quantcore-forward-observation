Duplicate prevention method:
- Monthly CSVs are append-only.
- New rows are skipped if an existing row has the same exchange, symbol, data_timestamp_utc, and source_name.
- For Bybit ticker snapshots, where no native data timestamp is provided, data_timestamp_utc is the UTC hour bucket, e.g. YYYY-MM-DDTHH:00:00Z.
- Validation repeated the same-hour logger run after this fix: 57 duplicates prevented, 0 rows written.
- Note: two earlier validation rows were created before the hour-bucket fix. They were left untouched to respect append-only storage.

Timestamp handling:
- All timestamps are UTC ISO strings.
- Exchange millisecond timestamps are converted to UTC ISO when available.
- If no native timestamp is available, data_timestamp_utc uses the fetch hour bucket for duplicate safety.
- fetch_timestamp_utc records the exact UTC fetch time.

Error handling:
- Per-symbol failures do not stop the run.
- Per-data-type failures are logged in the row and health CSV.
- HTTP calls use limited retries with short backoff.
- No interpolation, forward fill, or synthetic values are created.
- If all endpoints fail, the logger exits nonzero after printing a JSON summary.

Health check behavior:
- scripts/e2_health_check.py reports rows collected in the last N hours, expected rows, coverage percentage by data type, missing symbols, error rows, duplicate count, longest observed timestamp gap if feasible, and latest successful timestamp.
- It writes data/derivatives_forward/health/e2_daily_health_report.txt and prints the same summary.
